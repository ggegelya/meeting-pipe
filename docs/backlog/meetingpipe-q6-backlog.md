## How to pick up a task (delegation)

Mechanics are codified in `/tech-task <ID>` (read the task here, read the orientation docs, implement, verify, one commit on `main`, do not push). Soft context that command does not carry:

- Task ids carry no `TECH-` prefix (e.g. `/tech-task STOR1`). Historical code comments, ADRs, and commits keep the old `TECH-` form as provenance.
- **Full specs for items carried from Q4 unchanged live in [`q4-final.md`](./q4-final.md)** (search the id); the Task specs section below restates the new and changed items and points to q4-final for the rest, so each spec has a single owner.
- User locale: 99% English / Ukrainian. Do not default examples to German. Verify any new `uk` labels in `MuteLabels.toml` with the user.
- The user is vibe-coding: high-level explanations, does not read code. Lean on the ARCHITECTURE.md diagrams. Blind sub-pixel visual churn is out of bounds; visual changes need eyes, and the daemon is a menu-bar app that is not screenshotable here, so a rendered eyeball is owner-owed.
- No em-dashes in any output (CI fails on a U+2014 in an added line under `daemon/Sources` and `daemon/Resources`). Use hyphens, commas, or rewrite. Commit with the repo's git identity. Do not push without permission.
- Binding constraint (CLAUDE.md): the primary user is the author; selling is tertiary; technical excellence in core functionality comes first. The BRAND launch band stays deferred. The 2026-07 redesign band (DSN20-29 + WF6 + UX15) shipped in Q5; the only redesign residual is the owner-run `/design-sync` push, an outward sync tracked as an owner action, not a task.
- The AI/DV standing rule: every `AI*`/`DV*` capability and any new engine call must route through the local engine path so the `egress_guard` clamps it under regulated/NDA, honour `effective_backend()` (forced local under regulated/NDA), and add no new always-on egress.

## Marking a task done

When a task ships, set its **Status** column to `done` (sadok-style) rather than deleting the row or prefixing the id, so the trail stays readable within the quarter; a shipped item moves to the next `q<N>-final.md` at the quarter roll. A partial ship is `partial` with the remainder named in the Comment; owner-owed runtime or eyeball work is `owner-owed`.

**AI6 (P2): recurring-series continuity.** Summaries of recurring meetings re-extract known context and lose the thread between sessions. Scope: at summarize time, inject the same workflow's previous meeting as a bounded context block (its decisions plus its still-open actions, post-AI1 resolved state), with instructions to mark carried-over items as carried over rather than re-extracting them as new discoveries. The workflow is the series key (explicit, already resolved per meeting); no fuzzy series inference. Prompt-only in v1, no schema change; the block is bounded (~1K tokens, latest prior meeting only) and assembled from disk, so cloud and local backends get it identically and the egress profile is unchanged. The first meeting in a workflow is a clean no-op. Acceptance: the second meeting in a workflow produces a summary that acknowledges the carried-over open action instead of re-listing it as new (dogfoodable via `mp dogfood`); regulated/NDA behaviour unchanged.

**AI4-FINISH (P2): digest surfacing, schedule toggle + Digests view.** AI4 shipped CLI-only with scheduling left to hand-written cron, exactly the CLI-no-UI pattern the Q4 validation audit flagged on FEAT2/FEAT4. If it lands before DSN27, a plain rail row is fine and the INSIGHTS grouping arrives with the port. No new egress (the digest already routes through the engine + clamp). Acceptance: the toggle installs a LaunchAgent that fires `mp digest` at the configured time; digests render in the Library; disabling removes the agent; visual eyeball owner-owed.

### Feature review 2026-07 (STOR/PROV/AUTO bands, ASR1, and additions across existing families)

Origin: an owner-commissioned review of the shipped surface against this backlog (2026-07-05). The finding: correctness, detection, and design are well covered, and the gaps cluster in **archive stewardship** (the product accumulates an irreplaceable corpus, corrections, voiceprint, roster, library, with no storage, backup, or sync-safety story) and in **half-closed loops** (corrections advertise a LoRA phase no task owns; digest and long-meeting flows dead-end in CLI or manual steps). The owner approved all twenty review items on 2026-07-05 and proposed the provider seam (PROV1) in the same pass. Bucket mapping: must to P1, should to P2, nice to P3 (P3 here means promote on demand, per the AI5/I8 precedent). The standing AI/DV rule applies to every item that touches an engine.

**STOR1 (P1): audio library storage governance.** Stereo WAV is ~0.7 GB per hour and `raw/` grows unbounded; MIC13 covers only the `originals/` recovery copies and PERF7 only the event logs. Scope: a per-workflow audio retention policy (keep forever / transcode to FLAC or AAC N days after a terminal published-or-skipped state / drop audio keeping transcript + summary), a reaper sweep sharing MIC13's mechanism (one reaper, two scopes), a Preferences Storage section (library size, per-policy breakdown, HuggingFace model-cache size with an evict-unused-models control, waveform-cache size), and disk numbers in `mp doctor`. Consumers must survive a transcode: playback and the waveform peaks re-derive from the new file (the cache is keyed on size + mtime, so it self-invalidates) and the Library's audio affordances follow the extension. Defaults are conservative: every workflow starts on keep-forever until the owner opts it in, and a policy never touches a non-terminal meeting (`Needs you` members are exempt). Acceptance: a meeting past its window is transcoded or reaped on the next sweep with playback and waveform intact; delete cascades stay correct against MIC13; Preferences and doctor report real sizes; keep-forever workflows are byte-unchanged.

**SEC12 (P1): iCloud/cloud-sync egress check on the library path.** The default `~/Documents/Meetings/` sits inside iCloud's Desktop & Documents scope when that setting is on, so every WAV and transcript silently uploads to Apple, an OS-level hole in the zero-egress claim; ADR 0016 already mandates iCloud/Time-Machine exclusion for `originals/`, but the main library has no check. Scope: detect a cloud-synced library root (iCloud Desktop & Documents state, plus Dropbox / Google Drive / OneDrive path patterns), WARN in `mp doctor` and in a Preferences Storage status line naming the provider, and hard-FAIL the doctor check under `regulated_mode` or when any NDA workflow exists; offer an assisted move to a non-synced root (guided, never performed without explicit confirmation) or a per-folder exclusion where the provider supports one. README documents the caveat. Acceptance: a synced library path is flagged with the provider named; regulated + synced is a FAIL, not a note; the assisted move yields a working library at the new root with config updated.

**ASR1 (P1): post-ASR glossary / custom vocabulary.** Recurring proper nouns, client and product names, and en/uk code-switched terms are mangled by ASR every meeting, and the damage propagates into summaries, embeddings, ask, and roster display. Scope: a deterministic substitution layer applied at transcript finalize in the pipeline, before summarize/embed ever see the text: a global glossary plus an optional per-workflow table (TOML under `~/.config/meeting-pipe/`, the workflows precedent), case-insensitive whole-word matching that is Unicode-aware (Cyrillic word boundaries), longest-match-first, with applied-substitution counts logged to pipeline events. Transcripts stay write-once: the glossary applies to new meetings at finalize; re-applying to an old meeting is an explicit regenerate, never a mutation. An optional bounded-edit-distance fuzzy stage for names ships behind a flag, off by default (a wrong substitution is worse than a missed one). Any new `uk` entries are verified with the user (the locale note above). Acceptance: a glossary term is normalized consistently across transcript, summary, and the embedding index; per-workflow entries apply only to that workflow's meetings; zero egress impact (a pure local transform).

**FEAT8 (P1): in-meeting moment flag.** The one in-meeting capability the competitor set has that meeting-pipe lacks, and it fits the register: one global hotkey (configurable, chosen in-task alongside `manual_hotkey`) during a recording stamps a timestamp marker; the HUD acknowledges with a single quiet blink (opacity-only, reduced-motion honoured), no persistent chrome. Markers are written at stop as `<stem>.markers.json` (schema_version stamped, documented in CONVENTIONS, the DOC6 precedent); the transcript tab renders each marker as an anchor chip at its timestamp (click = seek, the existing play-head path); summarize receives the flagged excerpts (the segments spanning each marker) as user-flagged moments with an instruction to reflect them, so capture is LLM-free and deterministic while emphasis is model-side. BYO and long-meeting paste bundles include the marker list. No new egress class: flagged excerpts travel to whichever backend already summarizes the transcript. Acceptance: a hotkey press mid-recording lands a marker in the sidecar; the chip seeks correctly; a flagged moment demonstrably surfaces in the summary; works under regulated/NDA; visual eyeball owner-owed.

**STOR2 (P1): backup + restore manifest.** The compounding assets are irreplaceable and scattered: the library (`raw/` + sidecars + `digests/`), `~/.config/meeting-pipe/` (config, workflows, voiceprint, roster, glossary), the corrections corpus in `~/Library/Application Support/MeetingPipe/`, and the Keychain items; caches are rebuildable and excluded. Scope: `mp backup <dir>` writes a dated archive of the non-rebuildable roots plus a manifest naming the Keychain items to re-create (secret values are never exported); `mp restore <archive>` unpacks to the configured roots (per ADR 0003 a manual copy also works, and the runbook says so); `mp doctor` reports last-backup age when a backup marker exists; a README migration runbook (new Mac: install, restore, re-add the Keychain items, done). Acceptance: a backup restored into a scratch HOME yields a working library, workflows, roster, and corrections state; the manifest names every Keychain item; doctor reports backup age; the runbook is verified by walking it.

**MIC14 (P2): off-the-record manual redaction spans.** Mute redaction only protects spans where the meeting app was muted; the human cases (a side conversation, an interruption, "this next part stays out") have no affordance. Scope: a hotkey toggles an off-the-record span mid-recording; spans are recorded into the existing mute-timeline machinery tagged `source: manual` (`<stem>.mute-timeline.json` grows a per-span source field, version-bumped and documented), and MuteRedactor redacts them in the transcribed, summarized, and published artifact while the full recording stays local for recovery, the ADR 0016 posture. Two invariants: a manual span is explicit intent, so it is always redacted and exempt from MIC12's speech-bearing withhold; and under the regulated/NDA real-time gate path the hotkey zero-fills live instead (no unredacted audio at rest there). The HUD shows a truthful persistent state while active ("Off the record") so it cannot be forgotten, and an open span auto-closes at stop. Acceptance: span audio is absent from transcript, summary, and every published artifact and present in the local original (capture-first path); the HUD state is truthful; the timeline round-trips; MIC12's guard leaves manual spans alone.

**LOCAL9 (P2): corrections Phase 3, local LoRA fine-tune.** `mp corrections-stats` advertises a Phase 3 readiness check (~20 corrections over ~200 minutes) that no backlog task owns; make it real or stop advertising it. Scope: an owner-invoked `mp train-adapter` that fine-tunes a LoRA on the standard 7B via the MLX LoRA path, building prompt/completion pairs from the corrections corpus (original vs corrected summaries); evaluation is `mp dogfood` A/B (base vs adapter) on held-out meetings, hand-graded like the engine comparison; the adapter ships only through an explicit config opt-in (`summarization.local_adapter_path`), never silently applied. Fully on-device, no egress; training is a command the owner runs, no background training. An honest terminal outcome is allowed: if the A/B shows no capture win, close as adapter-not-worth-it, keep the corpus as a prompt-context asset (few-shot exemplars are the cheaper fallback lever), and fix the README wording. Gated on the corpus bar, which is owner-owed grading volume. Acceptance: with the bar met, a training run produces a loadable adapter; the dogfood scorecard decides adoption; README's corrections section describes the real flow either way.

**FEAT3-SEGMENT (P2): per-segment speaker reassignment.** Roster naming fixes whole clusters, but a misattributed segment (diarization splitting or merging people mid-meeting) cannot be moved, and the README troubleshooting entry offers no user-side recourse. Scope: right-click a transcript segment, "Reassign to..." (me, a roster name, or unknown), applied through the FEAT3-ROSTER relabel path so the transcript, summary attendees, and later meetings stay consistent. v1 is a label fix only: no embedding is folded into any roster centroid on reassignment (a wrong fold poisons the centroid; enrollment stays the explicit "Name this speaker..." path), and the existing regenerate/republish affordances are offered afterward rather than auto-fired. Acceptance: a reassigned segment renders under the right speaker and persists across rescans; a subsequent republish reflects it; roster centroids are byte-identical after a reassignment.

**PIPE4 (P2): backend-aware long-meeting routing.** The 80k-char guard exists to protect the Anthropic bill, but it is backend-blind: `backend = local` meetings over the threshold dead-end in a paste bundle even though summarizing them locally is free (LOCAL3 confirms only Apple Intelligence is exempt today). Scope: make `skip_above_chars` gate the cloud backend only; when the effective backend is local (chosen, auto-resolved, or forced by regulated/NDA), a long transcript routes through a hierarchical map-reduce on the MLX path, generalizing LOCAL3's batched-reduce pattern out of `AppleIntelligenceSummarizer` into `summarize_local`, with Cyrillic-safe windows and the ~4K per-call context budget AI2 measured as safe (16K OOM-crashed the Mac). The paste bundle remains the cloud-over-threshold behaviour and the failure fallback, and the daemon watchdog stays the hard backstop. Acceptance: a >80k-char meeting on the local backend produces a summary end-to-end with no `READY_FOR_MANUAL`; the anthropic path is unchanged; a long Ukrainian meeting does not overflow the window (the LOCAL3 finding); memory stays bounded on the real Mac (owner-eyeballed, the VALID1 precedent).

**UX18 (P2): in-app paste completion for BYO / long-meeting bundles.** Completing a paste-pending meeting today means hand-saving `<stem>.summary.md` next to the transcript and running `mp publish-from-paste`, the only core flow that leaves the app. Scope: the paste-pending detail pane gains "Paste summary...", opening a sheet (raw markdown box, parsed through publish-from-paste's existing parser into the typed model, with inline validation errors); on confirm it publishes through the PIPE2 fanout with the workflow overlay applied and the egress guard armed, exactly like the CLI path, which stays supported. DSN22's "Reveal bundle" verb remains the manual escape hatch. Acceptance: a paste-pending meeting goes to published entirely in-app; an NDA workflow's paste still drops Notion (the PIPE2 behaviour); malformed input surfaces inline instead of failing silently; visual eyeball owner-owed.

**WF8 (P2): post-hoc workflow reassignment.** Matching happens once at record time and manual hotkey recordings always fall to the default, so a misrouted meeting is permanently stuck with the wrong context prompt, sinks, Notion DB, and NDA posture. Scope: "Change workflow..." in the detail `...` menu rewrites the `workflow_*` fields of `<stem>.meta.json` (the Swift writer; the CI2 golden-fixture contract catches drift), refreshes the row's chip and scope membership, and then offers the two follow-ups with plain-English consequences: Regenerate (the context prompt changed) and Republish (the sinks or DB changed). NDA transitions are the sharp edge and confirm explicitly: moving into NDA re-clamps everything from now on but cannot un-egress past publishes (the sheet says so); moving out of NDA is a deliberate downgrade and requires confirmation. Acceptance: a reassigned meeting republishes to the new workflow's sinks and DB; scope counts update; the meta contract fixtures gain the rewrite case; no automatic regenerate or republish without the user choosing it.

**PIPE5 (P2): Notion sink property enrichment.** The sink writes a three-property husk (Name/Date/Status) while attendees, workflow, duration, and the open-action count all sit in the sidecars, so the database the owner actually lives in cannot filter by any of it. Scope: probe the target database schema (reuse the workflow editor's cached `notion-databases.json` pattern) and fill matching optional properties by name and type when present, e.g. `Workflow` (select), `Attendees` (multi-select), `Duration` (number, minutes), `Open actions` (number), `Source` (select); fail-soft everywhere: an absent property is skipped silently, a type mismatch is skipped with one log line, and the sink never creates or mutates database schema. The README's schema table documents the optional set. Acceptance: a database carrying the optional properties gets them filled on publish and updated on republish (the upsert path); a bare Name/Date/Status database behaves byte-identically to today; the regulated no-op is unchanged.

**PROV1 (P2): pluggable cloud providers, claude CLI first, then OpenAI API.** Owner-proposed 2026-07-05. Today the backend set is anthropic / local / auto / apple_intelligence, each wired point-to-point. Two motivations: cost, a Claude subscription already pays for Claude Code, so spawning headless `claude -p` summarizes at zero marginal API cost (and, with PIPE4, removes the long-meeting bill entirely on that backend); and flexibility, a BYO OpenAI-compatible key. Scope, in order: (1) formalize the provider seam over the two existing call shapes, structured summarize and `engine.complete_text`, so a provider is one module implementing both; (2) `claude_cli`: spawn `claude -p --output-format json` with tools and MCP disabled, non-interactive, parsed and validated through the existing pydantic path, with `mp doctor` preflighting binary presence, auth state, and version; (3) `openai`: an API provider over httpx (so the egress guard clamps it like anthropic), key in the Keychain via the SEC8 pattern, model id in config; (4) codex CLI only if (2) proves the CLI shape stable. The load-bearing sharp edge: a CLI provider egresses through a child process, outside the in-process httpx egress guard, so `effective_backend()` must classify every CLI provider as cloud and refuse it under regulated/NDA fail-closed at the policy layer (the SEC10 posture), with tests asserting the CLI is never spawned in those modes; `auto` keeps its anthropic-then-local ladder and never falls back into a CLI provider silently. This amends the README's "Messages API directly, not Claude Code" rationale, which predates headless `-p`; record the amendment (a short ADR or a note on ADR 0007). Acceptance: `backend = claude_cli` summarizes end-to-end with no API key configured; doctor preflights it honestly; regulated/NDA never spawn a CLI; the openai provider passes the same egress-guard tests as anthropic; the Preferences backend picker gains the entries.

### Local models and performance (relocated from the Q5 ToC)

**LOCAL7 (P2): local summaries honor summary_language on non-English meetings.** The engine comparison found the local backends ignore `summary_language = auto` on Ukrainian: the 7B drifts uk to en, the 14B uk to ru; only Anthropic preserves Ukrainian. Strengthen the local language directive (`summarize_local` plus the Apple Intelligence instructions) and add a cheap detected-language-mismatch retry so a uk transcript yields a uk (or the configured-language) summary.

**LOCAL8 (P2): Apple Intelligence, experimental gate + viability fixes.** The engine comparison showed the macOS 26 Apple Intelligence backend is native/free/zero-egress but not usable as a daily summarizer today: its 4096-token context forces heavy chunking (minutes/meeting), it echoes raw dialogue on short meetings, ignores the bullet cap (~11 vs 5), mislabels Ukrainian, and 1/5 failed. Keep the code but mark it experimental / not-recommended in Preferences (so it is not silently chosen), and track the fixes that make it viable: LOCAL3 (reduce + Cyrillic window, shipped in Q5), LOCAL7 (language), bullet-cap adherence, the short-meeting transcript-echo. Revisit-when: those land OR Apple widens the system-model context. Owner kept it (not dropped) as the only native zero-egress path and the I7 trigger.

**PERF6 (P3): gate discovery-scan AX walks behind a cheap precondition.** The deeper lever under the 2026-06-30 idle-energy finding: Activity Monitor showed MeetingPipe spiking ~8-13% CPU while idle, and `sample` put the bursts in `MeetingSourceScanner.enumerateCandidates` doing three recursive AX tree walks per running meeting app (`MeetingAXHandleBuilder.findToolbar`/`findButton`, dominated by `mach_msg2_trap` AX IPC). The fixed 3 s discovery backstop re-walks Teams' large AX tree every 3 s all day even when no meeting is plausible; PERF5's backoff only ever covered the in-meeting 1 Hz pollers, never discovery. Fix (kept separate from PERF5, whose remainder is the unrelated 4 Hz engine-tick gating): in `collectSignals`, gate the three button walks (`findCallingControlsToolbar` + `findAllLeaveButtons` + `findAllMuteButtons`) behind a cheap precondition (a meeting-titled window from `collectAXWindowTitles`/`isActiveMeetingWindow`, or active process audio), so an idle backgrounded meeting app skips the expensive walk. Changes signal-collection semantics, so validate: `DetectionCorpusTests` green and a real meeting still scores (dogfood / `mp analyze-detection`). Pairs with the lighter first-line fix, adaptive backoff on the discovery backstop poll.

**PERF7 (P3): event-log rotation / size cap.** `events.jsonl` grows unbounded (63 MB / 338k lines on the dogfood Mac), dominated by the 1 Hz `ax_leave_button_state` + mute-state events written for the whole of every meeting; no rotation or trim exists. Add a size- or age-based rotation (or ring-buffer trim) for `events.jsonl` / `pipeline_events.jsonl` so the logs self-bound while `mp logs` / `mp analyze-detection` keep working on the recent window. Pure hygiene (per-meeting disk-write energy + unbounded growth), no detection change. Sibling to SEC11's `events.jsonl` `0644`/gitignore hygiene, a different fix (mode vs size).
