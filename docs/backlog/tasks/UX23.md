# UX23: Shipped-surface floor

Band origin: assessment review 2026-07-12. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**UX23 (P2): shipped-surface floor.** Rough edges on recently shipped surfaces the owner has not yet e2e-tested, each violating a written floor or the quiet-register promise. (a) The prompt panel is mouse-only: `PromptRecordButton` is a bare NSControl with manual mouseDown tracking, no `acceptsFirstResponder`/keyDown, and the close/chevron controls lack accessibility labels, contradicting PRODUCT.md's explicit keyboard + VoiceOver floor for exactly these buttons; add focus, Space/Return activation, labels, and a sane tab order. (b) Off-the-record (MIC14) is undiscoverable: no Preferences hotkey row (the other three hotkeys have one), no HUD affordance; add the SettingsHotkeyField row and a HUD toggle where the moment happens. (c) Search cannot distinguish "index still building" from "SQLite failed to open" (`SearchIndexer.matchingStems` returns nil for both and silently degrades forever); publish an indexing/degraded state, show a one-line hint in QuickFind and the filter bar, and add a search-index probe to the doctor sheet. (d) DigestsView's Generate now discards its `Result`, so a failed `mp digest` renders nothing; capture into an inline error row (AskView.errorState is the in-file pattern). (e) The status-bar permission warning does not name consequences; per missing permission, say what stops working ("Screen Recording off: browser meetings will not be detected"). Acceptance: prompt panel fully keyboard-operable with labels; the off-record hotkey is visible and editable; QuickFind shows an indexing hint and doctor reports index health; a failed digest shows why; the permission warning names the consequence.
