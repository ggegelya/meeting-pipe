# REC8: Self-healing capture

Band origin: assessment review 2026-07-12. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**REC8 (P2): self-healing capture.** Mid-meeting system-audio death recovery is gated on the owner noticing a HUD banner and clicking Retry (REC4 deliberately chose manual; the plumbing exists: `handleSystemAudioLost`, `retrySystemAudio`, `onSystemAudioRecovered`); the daemon holds no power assertion, so an in-person manual recording with no meeting client can idle-sleep the Mac mid-recording; there is no wake hook, and the tap-liveness watchdog is gated on `engine.isRunning`, so a silently stopped engine is never re-armed. Scope: one automatic bounded retry ~5-10 s after a system-audio loss, then fall back to the banner (preserves REC4's partial-recording rationale); `IOPMAssertionCreateWithName` (prevent idle sleep) across recording, released in `stop()` and the termination flush; observe `NSWorkspace.didWakeNotification` and re-arm or fail cleanly if recording; fix the watchdog gate. Acceptance: a killed SCStream recovers hands-off on the first retry; `pmset -g assertions` shows the assertion during recording and not after; a sleep/wake mid-recording ends with a usable artifact.

**Progress (2026-07-12): implementation shipped + unit-tested; the live-Mac acceptance is owner-owed.** All four legs are in `MeetingRecorder`: (a) `scheduleSystemAudioAutoRetry` fires one `retrySystemAudio` ~7 s after `handleSystemAudioLost`, then falls back to the manual banner (one per loss, cancelled on stop / termination / a manual retry); (b) `IOPMAssertionCreateWithName(kIOPMAssertPreventUserIdleSystemSleep, ...)` is held from `start()` and released in `stop()` + `flushIntermediatesForTermination()` (best-effort, logged, never a thrown start); (c) an `NSWorkspace.didWakeNotification` observer re-arms the mic tap if the engine died over sleep and retries system audio, registered in `start()` and removed in stop + termination; (d) the tap watchdog is un-gated from `engine.isRunning` via a new pure `TapWatchdogPolicy.decide` (a stopped engine now re-arms regardless of the buffer counter, the exact case the old gate disabled), routed through `checkTapLiveness`. New `recorder` events: `system_capture_auto_retry`, `idle_sleep_assertion_held`/`_released`, `did_wake_while_recording`. Verified: 1280 daemon green incl. `TapWatchdogPolicyTests` (3); no docs changed (all new events use the existing `recorder` category and no surface made a now-stale claim). Owner-owed remainder (needs a real Mac, not exercisable here): confirm `pmset -g assertions` shows the assertion during a recording and drops after, that a killed SCStream recovers hands-off on the first auto-retry, and that a sleep/wake mid-recording ends with a usable artifact.
