# UX26: Resolve action items from the Summary tab

Band origin: filed 2026-07-22 (owner feature ask, approved in the 2026-07-22 review session). Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**UX26 (P2, new): mark an action item done (or reopen it) directly on the Summary tab.** The lifecycle already exists end to end except the affordance on the surface the owner actually reads. `ActionItem.resolved` is the AI1 state (pipeline/src/mp/schemas.py:28); the Facts rail's "Mark done" writes it straight into `<stem>.summary.json` (FactsView.swift:141-168: optimistic removal, captured-index-with-text-match guard, atomic write, `action_resolved` library event, deliberately no correction record); the Notion publisher round-trips it as a checked to-do (publish_notion.py:663-667); DSN2's needs-republish badge lights when the summary is newer than the last publish, so republish needs zero new logic. The gap, and a truthful-states gap (PRODUCT.md principle 2) besides: the Summary tab renders resolved and open actions identically, because the renderer drops the flag entirely (`SummaryRenderedView` maps only task/owner/due/confidence, SummaryRenderedView.swift:79-88).

Scope: extract the FactsView.markDone write into one shared helper (home: `Library/MeetingLibraryService.swift`, the existing library mutation service) taking (summaryURL, actionIndex, taskText, resolved) with the same guard (prefer the captured index while its task text still matches, else the first text-match with the opposite state), atomic rewrite via `MeetingSummary.jsonObject()`, events `action_resolved` / new `action_reopened` (existing `library` category, snake_case); FactsView and the Summary tab both call it. `SummaryRenderedView`'s ActionItemRow gains the resolved state plus an optional toggle callback: nil keeps today's static render for DigestsView, the A16 candidate/compare pane, and any surface with no live summary.json behind the rows; SummaryTab passes the callback for the live summary only. UI: the static circle glyph (SummaryRenderedView.swift:157) becomes a small always-visible toggle, circle when open, filled checkmark when resolved; a resolved row stays in place with the task struck through and dimmed, chips dimmed, reversible with the same click; no row removal, no layout jump. A11y: VoiceOver "Mark done" / "Reopen" naming the task, Tab-reachable, state carried by icon + strikethrough, never color alone. Reload interplay: the rewrite bumps `MeetingStore.revision` and SummaryTab's mtime-guarded reload picks it up (SummaryTab.swift:509-531); the toggle updates optimistically first, the FactsView pattern.

Explicitly not: no "remove as noise" affordance (deleting a wrong action already exists via Edit summary, CorrectionEditor.swift:136-138, and staying there keeps the LOCAL9 correction corpus fed); no correction record on a toggle (a state flip is not a text correction, the FactsView precedent); no auto-republish (the needs-republish badge is the nudge, DSN2's single-republish-path decision); no cross-meeting dedupe or overdue semantics (AI7 owns that). Interlock with FEAT10: whichever lands second must keep the shared writer preserving fields it does not itself edit (the evidence field once Swift models it).

Acceptance: shared-helper unit tests pin the index-guard fallback on a shifted summary, the reopen path, preservation of untouched fields (title, extra_sections, and evidence when present) through a toggle, and both events. A renderer test pins resolved to the struck/dimmed presentation and callback-nil to the static render. Behavior: a Summary-tab toggle flips `resolved` in `<stem>.summary.json`, the row shows the struck state immediately and after the store rescan, Facts drops the item on its next aggregation, a republish ticks the Notion to-do with zero new publish code, and reopen reverses all of it. swift test, ruff, pyright, pytest, and truth fences green (the new `action_reopened` action is snake_case in the existing `library` category). Owner-owed eyeball: the toggle and strikethrough on a real meeting.
