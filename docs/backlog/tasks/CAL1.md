# CAL1: Spike: EventKit calendar context

Band origin: carried from Q4; base spec in q4-final.md, progress notes here. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**CAL1 (P2, partial): spike EventKit calendar context (re-evaluates ADR 0011).** Base spec in [`q4-final.md`](./q4-final.md) (search the id). **Progress (2026-07-12): analysis + measuring instrument shipped; the empirical verdict is owner-owed.** The spike doc [`docs/spikes/cal1-eventkit-calendar-context.md`](../spikes/cal1-eventkit-calendar-context.md) reframes the question against ADR 0011: ADR 0011 rejected calendar as an end-detection *corroborator* (telemetry, no verdict impact, no consumer) and leaned on an *unmeasured* assumption ("this user's meetings are not reliably calendar-bound"); CAL1 evaluates calendar as **hints only, never a trigger** (pre-title the prompt + sidecar, feed the covering event's scheduled end into `IdleStopBackstop`'s horizon, preflight-warn a broken capture precondition before a scheduled meeting) and measures the assumption. The unavoidable cost is a new Calendar TCC prompt, at odds with the reduce-first-run-prompts direction, so a GO is only justified above a coverage bar. The owner-run, read-only probe `daemon/scripts/cal1-calendar-probe.swift` (compiles clean; not exercisable here, no calendar/TCC) enumerates recorded meetings (stems carry their start time) and reports, per meeting, whether a covering EventKit event exists (±tolerance) and whether the titles resemble, then a coverage % and a READ line mapping the % to GO (>=~60%: superseding ADR + build pre-title first) / marginal (~30-60%: maybe pre-title only) / close (<~30%: confirm ADR 0011 empirically). Provisional lean, pending the number: pre-title is the one hint worth the permission at moderate coverage; the horizon + preflight need high coverage. Owner-owed remainder, the spike's whole point: run `swift daemon/scripts/cal1-calendar-probe.swift` on the real calendar and decide GO-or-close from the coverage it reports. Synergy noted: a covering event's attendee list is a clean-attendee seed for FEAT3-ROSTER / the DV2 pivot (a separate follow-on spike, not a freebie).
