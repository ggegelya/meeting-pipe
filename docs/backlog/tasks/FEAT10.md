# FEAT10: Evidence-grounded summaries (citations with transcript jump)

Band origin: filed 2026-07-22 (owner feature ask, designed and approved in the 2026-07-22 review session; owner calls from that session: published notes carry timestamps only, and the decisions section is included). Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**FEAT10 (P2, new): every summary point carries verified evidence (verbatim phrase + timestamp) and a one-click jump to the transcript row.** The ask: a summary bullet, decision, action item, or open question is currently uncheckable against the meeting; the reader either trusts it or replays audio. Give each point up to two evidence anchors, each a short verbatim phrase plus a timestamp, rendered as quiet [mm:ss] chips in the Summary tab (hover shows the phrase) that switch to the Transcript tab, scroll to the source row, and briefly highlight it. Published Markdown gets plain ` [12:34]` suffixes, timestamps only.

Why the codebase is ready (evidence, line numbers as of 2026-07-22): the transcript sidecar carries absolute per-segment times (`TranscriptSegment` start/end, `daemon/Sources/MeetingPipe/TranscriptTab.swift:10`); time-to-row mapping plus scroll already exist for FEAT8 markers (`TranscriptSegment.index(at:in:)` TranscriptTab.swift:28, `ScrollViewReader` + `.id(seg.index)` + `proxy.scrollTo` at TranscriptTab.swift:535/601/615); tab switching is one write to `MeetingDetailView.selectedTab` (MeetingDetailView.swift:15); and `mp ask` is the verification precedent (model cites, `rag.verify_citations` checks against ground truth, graceful fallback when a weak model cites badly, ask.py:103-110). The one real gap: the model input has no timestamps. `summarize.summarize` reads the overlaid speaker-merged Markdown (summarize.py:396) built by `transcript_corrections.overlaid_markdown` (applies the speaker + text overlays to the `<stem>.json` segments, then `render_markdown`, transcript_corrections.py:82-106), and `render_markdown` emits no times (markdown.py:26). Second hazard, named so it is not rediscovered the hard way: Swift rewrites `summary.json` by re-serializing only the fields it models (`MeetingSummary.jsonObject()`, `daemon/Sources/MeetingPipe/Library/MeetingSummary.swift:143`), from both `FactsView.markDone` (FactsView.swift:145) and `SummaryTab.persistEdit` (SummaryTab.swift:632), so an unmodeled evidence field would be silently dropped on the first mark-done or edit. Swift must model the field.

Scope, pipeline: (1) additive schema, ADR 0014 stays intact: `Evidence {quote: verbatim <=15 words, t: float seconds into the recording}`; `ActionItem.evidence: list[Evidence]` inline, and `MeetingSummary.evidence: {"summary"|"decisions"|"questions": list[list[Evidence]]}` index-aligned for the string sections; at most 2 anchors per item; legacy files (field absent) round-trip unchanged and every existing consumer ignores the field. (2) model-facing shape: `SUMMARY_TOOL`'s string sections become `{text, evidence[]}` objects so anchors attach at generation and can never misalign (schemas.py:77); one shared normalizer flattens the model shape to the stored shape and is used by every backend (the anthropic tool path in summarize.py; the JSON recovery paths in summarize_local.py, provider_claude_cli.py, provider_openai.py via `largest_balanced_json_object`). (3) timestamped model input: a timestamps variant of the overlaid render prefixing each merged turn with [mm:ss]; the stored `<stem>.md` is unchanged, and a meeting with only plain md (BYO paste) simply gets no evidence, no error. (4) verification pass in `summarize` after the language check: t must land inside or within 45 s of a real segment (snap t to that segment's start); the quote must be a normalized substring (case/whitespace/punctuation tolerant) of transcript text in that window; drop what fails and emit a pipeline event with kept/dropped counts. A wrong anchor never reaches disk. (5) LANG1 interplay: quotes are verbatim transcript language, so `summary_language.divergent_sections` must exclude evidence or a translated summary (summary_language=en over a uk transcript) gets falsely repaired. (6) published render: `render_summary_md` appends ` [12:34]` (multiple: ` [12:34, 18:22]`) per item (markdown.py:96); the Swift mirror `AppleIntelligenceSummarizer.renderMarkdown` matches, and the CI3 golden `summary-md-golden.json` is regenerated with an evidence-carrying case (`cd pipeline && uv run python ../scripts/gen_contract_fixtures.py`).

Scope, daemon: (7) `MeetingSummary` models Evidence with tolerant decode and `jsonObject()` serialization; the corrections editor remaps on save: an item whose text is unchanged keeps its anchors (text-match, the FactsView.markDone guard pattern at FactsView.swift:150-157), an edited or new item drops them, and index alignment for the string sections is rebuilt (the editor edits and deletes rows in all four sections, CorrectionEditor.swift:17-137). (8) `SummaryRenderedView` renders the chips after each bullet/decision/question and inside the ActionItemRow chip row (the existing chip idiom, SummaryRenderedView.swift:211): monospaced digits, muted tint, no sparkle per PRODUCT.md; hover shows the quote; VoiceOver reads "Source at 12:34: <quote>"; chips are keyboard-focusable; the target-row flash honors reduced motion. (9) a chip click sets `selectedTab` to transcript and hands TranscriptTab a pending seek time, consumed via the existing time-to-index mapping + scrollTo. This pending-seek seam is also what AUTO2 rides.

Backends and degradation: full treatment on anthropic, openai, claude_cli, and the local single-pass path. The local map-reduce path (`summarize_local._batched` / `_render_for_reduce`, summarize_local.py:110-124) and the Apple Intelligence backend (`Summarization/AppleIntelligenceSummarizer.swift`) emit no evidence in v1: absence over wrong anchors, and the verifier enforces the same rule everywhere. Old meetings gain evidence per-meeting via the existing Regenerate flows; no bulk migration. Expect a modest output-token bump (capped quote length and 2 anchors per item keep it small on cloud; on local it is the price of the feature).

Explicitly not: no quotes in published notes (owner call 2026-07-22; the phrase is hover detail in-app). No meetingpipe:// links in notes (AUTO2, filed separately at P3). No audio-seek on chip click in v1 (cheap follow-on once the pending-seek seam exists; `PlaybackBar` already lives in the Transcript tab, note for the pickup session). No evidence on `mp digest` / DigestsView (cross-meeting synthetic summaries; the renderer tolerates absence) or on WF7 extra_sections. No new config knob; the caps are constants. Rejected during design: inline [mm:ss] markers inside the bullet strings (pollutes Facts rows, `mp actions`, digest, the embedding index, and the corrections corpus) and a `summary: list[str] -> list[object]` migration (breaks the ADR 0014 lock and every publisher for no gain over the additive field).

Acceptance: Python tests pin the normalizer (all four sections), the verifier (valid anchor kept with t snapped; fabricated quote dropped; out-of-range t dropped; tolerant quote matching), the `render_summary_md` suffix, the LANG1 exclusion, per-backend evidence survival through JSON recovery, and that a legacy no-evidence summary round-trips unchanged. Swift tests pin decode/jsonObject round-trip preserving evidence through a mark-done and a correction save, the remap-on-edit rule, the chip presentation model, and the pending-seek time-to-row mapping. CI3 fixture regenerated and both suites green on it. End to end: a regenerated meeting's `summary.json` carries verified anchors and every rendered chip resolves to a real transcript row; ruff, pyright, pytest, swift test, and truth fences all green. Owner-owed eyeball: a chip click lands the Transcript tab on the cited row on a live meeting.
