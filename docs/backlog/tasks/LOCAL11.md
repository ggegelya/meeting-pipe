# LOCAL11: Warm-server identity verification

Band origin: assessment review 2026-07-12. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**LOCAL11 (P2): warm-server identity verification.** Warm `mlx_lm.server` reuse never verifies what the server serves: `_is_healthy` checks only HTTP 200 on `/v1/models` (`summarize_local.py:572-577`) and `--adapter-path` exists only at spawn, so a `local_model` or `local_adapter_path` config change silently serves stale weights, the run sidecar records the configured (not served) model, and a warm base-model server means the LOCAL9 adapter never actually serves. Scope: compare the `/v1/models` payload (plus an adapter field added to the LOCAL10 ownership marker) against the configured identity and respawn on mismatch when we own spawning; restart the daemon's `LocalModelPreloader` on the relevant `ConfigStore.didPersist`; a doctor probe reporting served-vs-configured mismatch (also a precondition for trusting the LOCAL9 A/B). Acceptance: flipping the model or adapter takes effect on the next run without a manual kill; doctor names a mismatch; the run sidecar records the truly served identity.
