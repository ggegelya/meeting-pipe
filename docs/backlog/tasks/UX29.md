# UX29: NDA-aware re-summarize affordance

Band origin: filed 2026-07-23 (owner finding, 2026-07-23 assessment session). Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**UX29 (P3, new): the UI offers a cloud action that can never do what it says on an NDA meeting, then stays silent about the substitution.** On 2026-07-23 the owner clicked "Re-summarize with Anthropic" twice on an NDA-workflow meeting (stem `20260723-103118`, toolbar events 07:46:26Z and 07:48:43Z), got local Qwen output both times, and read it as a bug. It is not: the daemon passes `--backend anthropic` and the pipeline's `effective_backend` clamps to local (the PIPE6 one-shot override deliberately "still flows through the regulated/NDA clamp"; the daemon additionally strips the API key from the child env, `PipelineLauncher.cloudSecretPolicy`, PipelineLauncher.swift:827-857). The clamp is the product's core privacy promise and stays untouched. The defect is purely communicative: the menu item is emitted unconditionally (`MeetingDetailView+Header.swift:205-208`, the comment at :203-204 even names the reliance on downstream clamping), and no surface says "this ran locally because NDA".

Scope. In the detail `...` menu (and any twin affordances), when the meeting's meta carries `workflow_nda_mode = true` or global regulated mode is on: render the cloud option disabled with an inline caption in the quiet register ("NDA workflow: summaries stay on this Mac"), and add a "Change workflow..." row beneath it that opens the existing reassign flow (`beginReassign`, MeetingDetailView+Header.swift:416-449), whose explicit NDA-transition confirmation is already the designed escape hatch the owner ended up using by hand. Derive the flag from the same meta the row already loads (MeetingStore carries the workflow block); no new state. VoiceOver reads the disabled reason.

Explicitly not: no per-meeting cloud override under NDA (it would weaken the promise the clamp exists to keep; the reassign flow with its confirm IS the override); no pipeline changes; the post-run provenance label ("Summarized locally") is PIPE11's run.json fix rendered by the existing badge, not new UI here; regulated-mode handling beyond reusing the same disabled state.

Acceptance. A presentation-model test pins: cloud item disabled with the caption when meta says NDA (and when regulated is on), enabled otherwise; the Change-workflow row routes to the reassign entry point; VoiceOver label carries the reason. Full daemon suite green. Owner eyeball: on an NDA meeting the menu explains itself and the path to Anthropic is one click away, with the existing confirmation intact.
