# AI10: Digest action list is unbounded

Band origin: assessment review 2026-07-12. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**AI10 (P2): digest action list is unbounded.** `mp digest` aggregates every still-open action across the whole library into one weekly digest: on the real 199-meeting library that is 970 action items in a single digest, past the point of review and, before the 2026-07-13 rendering fix, enough to freeze the reader. That fix is in the daemon (`SummaryRenderedView`'s action list is now a `LazyVStack`, and the digest reader moved from the narrow, vertically-split center column to the wide detail pane, `DigestReaderView`), so the acute symptom is gone; what remains is upstream. The digest should scope its action list to the review window (still-open commitments from the digest's period, not every open action ever), then cap, group (by workflow or meeting), and rank (recency + confidence/priority) what it shows inline, with the long tail reachable but not dumped. Complements AI7 (dedup collapses near-duplicate restatements across recurring meetings; this bounds the window and the count). Local-only, no new egress (the AI/DV rule): pre-processing of the action set the digest already computes on-device. Acceptance: a weekly digest over the full library renders a bounded, grouped action list scoped to its review window, with the raw count available but not the default inline dump.
