# UX21: First-run traps

Band origin: assessment review 2026-07-12. Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**UX21 (P2): first-run traps.** (a) Startup fires all four TCC dialogs before the OnboardingGate branch (`App.swift:148` runs `coordinator.start()` first; `Coordinator.swift:367` calls `requestPermissionsAtStartup()` ungated), so a fresh install gets four unframed system dialogs stacked over the onboarding window and the framed permissions step arrives pre-answered; gate the burst on onboarding completion and have completion trigger prewarm + refreshAll. (b) A local/NDA workflow never prefetches the MLX model: `ConfigRefreshCoordinator.swift:30-35` keys only on the global backend, and the onboarding "Client work (NDA)" preset creates backend .local with no cache check, so the first NDA meeting records fine and then summarize fails with a terminal-command remedy (a de-CLI violation in an error path); on workflow save and the preset, when the resolved backend is local and the model is uncached, show an inline "model not downloaded (~4.3 GB), Download now" affordance reusing ModelDownloadSupervisor, and teach doctor's local-stack check to also run when any workflow carries backend=local or nda_mode. (c) Onboarding's Accessibility row gives no relaunch hint even though the product's own model requires one (the Preferences footnote documents it). Acceptance: a fresh install shows zero TCC dialogs before onboarding's permissions step; saving a local-backend workflow with no cached model offers the download inline and the failed-row remedy is in-app; the AX row explains the relaunch.
