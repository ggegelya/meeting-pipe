# AUTO2: Published-note transcript deeplinks (meetingpipe:// open at a timestamp)

Band origin: filed 2026-07-22 (owner feature ask, approved in the 2026-07-22 review session; P3 promote-on-demand by design: the in-app jump ships in FEAT10, this is the out-of-app half). Status and priority live in this task's ToC row in [meetingpipe-q6-backlog.md](../meetingpipe-q6-backlog.md).

**AUTO2 (P3, new): make the published [mm:ss] timestamps clickable from Obsidian / filesystem / LAN notes.** FEAT10 puts plain ` [12:34]` suffixes in published Markdown; this task turns them into `meetingpipe://open?stem=<stem>&t=<seconds>` links that open the Library at that meeting's Transcript tab at that moment. The scheme is shipped and hardware-verified (AUTO1, 2026-07-20, an `automation` `command` event with verb `open_library`): `AutomationCommand.parse` is the pure tested parser (Automation/AutomationCommand.swift:46) whose `library`/`open` verb already takes a scope, the router (`Coordinator+Automation.handleAutomation`) is the single gate, and `CFBundleURLTypes` is already registered by install.sh, so only new verb parsing lands in code. AskView already navigates from a citation to its stem (AskView.swift:12) and FEAT10 adds the pending-seek seam into TranscriptTab; this task is the URL spelling of those two existing moves.

Scope: parser support for `meetingpipe://open?stem=<percent-encoded stem>&t=<float seconds>` (t optional; a garbage t degrades to open-without-seek; bare `open`/`library` behavior unchanged), with cases in the AutomationCommand tests. Router: resolve the stem to a Library row (the AskView navigation precedent), select the Transcript tab, hand over the pending seek; an unknown stem (stale note, renamed recording) opens the Library quietly with no alert. Publishers: `render_summary_md` gains an optional deeplink mode (a stem parameter) rendering each timestamp as a `[12:34](meetingpipe://open?stem=...&t=754)` link, enabled for the Markdown-file sinks (publish_obsidian.py, publish_fs.py, publish_lan.py) and the stored `<stem>.summary.md`; Notion keeps plain-text timestamps (custom URL schemes are not reliably clickable in Notion rich text). The Swift mirror `AppleIntelligenceSummarizer.renderMarkdown` matches and the CI3 golden `summary-md-golden.json` gains a linked case.

Explicitly not: no Notion deeplinks (revisit only on evidence the scheme renders clickable there); no new UI for stale stems beyond the quiet no-match open; no install.sh changes (the scheme is already registered; a new verb is parse-side only). Depends on FEAT10 (the timestamps in notes and the pending-seek seam); promote to P2 once FEAT10 ships if the owner wants the out-of-app half next.

Acceptance: parse tests pin the happy path, percent-encoded stems, missing and malformed t, and no regression on the shipped verbs (the MeetingPipeAppIntentsTests vocabulary fence stays green). A router-level test pins known-stem to Transcript-tab-with-seek-pending and unknown-stem to Library-without-seek. A republished Obsidian note carries the links and the Notion output is unchanged, both fixture-pinned; CI3 golden regenerated; both suites and truth fences green. Owner-owed: click a link in a real Obsidian note on this Mac and land on the cited transcript row.
