## Task specs

**[DONE] TYPE1 (P2): static type checking in pipeline CI.** Shipped 2026-07-10. Original scope in the ToC row above. pyright (basic mode) is a dev dependency and its own CI step, configured in `pyproject.toml`'s `[tool.pyright]` over `src/` only; `tests/` stays out of scope (its fakes are deliberately partial) and ratcheting to strict remains a later decision. `src/` is at 0 errors.

The 26 findings were nearly all real, not annotation noise. The headline ones: `diarize.resolve_speaker_labels`'s untyped `roster` param (named in the spec; now `RosterStore`, imported under `TYPE_CHECKING` because `roster` imports `cosine_similarity` back out of `diarize`); `engine.AnthropicTextClient.complete` filtering content blocks with `getattr(b, "type", None) == "text"`, which defeats the discriminated-union narrowing that a plain `b.type == "text"` gets, so `.text` was being read off ten block types that do not have it; `egress_guard`'s `_orig_sync` / `_orig_async` typed as `None` and then called (the wrappers now read the global at call time, which is what lets the tests stub them, and raise rather than call None); `apply_speaker_labels` passing a possibly-None speaker as a dict key; and `_duration_sec` narrowing `seg.get("end")` on a value it then re-fetched. Two small consolidations fell out: `summarize()` now returns a `SummaryOutput` TypedDict instead of `dict[str, Path | str]` (which is what let `orchestrate` drop a pile of defensive `Path(...)` / `str(... or "")` re-coercions at the call site), and `schemas.Confidence` names the `Literal["low","medium","high"]` that `digest._clamp_confidence` coerces onto.

Two things the spec did not anticipate. (1) pyright reads its interpreter from config, not argv[0], so a bare `.venv/bin/pyright` (how CI invokes ruff and pytest) reported *every* third-party import as unresolved; `venvPath` / `venv` in `[tool.pyright]` fixes it. This was caught by building a CI-identical light venv in a scratch dir rather than trusting the local run, where `uv run` had been exporting `VIRTUAL_ENV` and masking it. (2) CI deliberately never installs `mlx_lm` / `mlx-embeddings` (darwin/arm64 wheels only) and therefore not `huggingface_hub` either, so three lazy imports are unresolvable there. They carry a per-site `# pyright: ignore[reportMissingImports]` rather than a global `reportMissingImports = "none"`, so a genuine typo in any other import still fails the build. Acceptance verified end to end in the CI-identical venv: clean exits 0, an injected `render_summary_md(int)` exits 1.

One behaviour note: tightening `orchestrate`'s run-sidecar write to `s["backend"]` broke `test_orchestrate_tolerates_summarize_without_meta`, which pins that a summarize result missing those keys must not cost the sidecar. The tolerant `.get` was restored (a TypedDict is a static promise Python does not enforce, and that write is best-effort); the test's own docstring, which justified the tolerance by citing the byo and long-meeting paths, was corrected, since those short-circuit before the sidecar write and never reach it.

**[DONE] LOCAL10 (P2): local-engine server lifecycle, no orphans.** Shipped 2026-07-10. Original scope in the ToC row above. As built: `mp.local_server` owns a `~/Library/Logs/MeetingPipe/mlx-server.json` marker (pid, owner pid, port, model, spawn time, `0600`), written by `LocalSummaryClient._spawn` and cleared by `_terminate_proc` on every clean path. `mp doctor`'s new `check_local_server` reports an orphan without killing it; Swift's `LocalServerReaper` kills one from two triggers: `Coordinator.reapStorage()` at launch (the OriginalsReaper slot the spec named) and, better, `PipelineLauncher`'s termination handler immediately after the watchdog kills a job, which is the case that creates the orphan in the first place. The schema is documented in CONVENTIONS.md as a Python-to-Swift contract, and both sides gate on *process identity* (`ps -o command=` contains `mlx_lm.server`) rather than bare liveness, so a recycled pid is never signalled and a recycled owner pid cannot shield an orphan.

Two corrections to the spec's own premises, both verified in code before building anything. First, "the server is reused warm across runs and must survive its parent `mp` exiting" is not what `_spawn` does: `summarize()`, `engine._complete_local`, `_AutoFallbackClient`, and `diarize_cleanup` all call `close()`, which terminates the server, so a `_spawn`ed server never outlives a *clean* `mp`. The real warm-reuse path is `mp serve-local`, which the daemon spawns directly and owns through a child handle, and which therefore can never be orphaned. `setsid` buys signal isolation, not warm reuse, and the orphan window is exactly "the parent was killed before its `finally` ran". Second, and following from that, spec part (c) is a **NO-GO**: `serve-local` `execvp`s into `mlx_lm.server`, so no idle timer can survive the exec; keeping one would require a supervisor process (the layer `serve-local`'s design comment explicitly rejects) which additionally could not observe the HTTP traffic it would have to call idle. The marker plus a reaper owned by the daemon, the one process that outlives every `mp` and is the one doing the killing, is the right shape. Acceptance met: a watchdog SIGKILL now reaps the server within seconds rather than "beyond the idle window", the next launch reaps anything missed and emits `pipeline`/`local_server_reaped`, warm `serve-local` reuse is untouched (it writes no marker), and doctor reports honestly, including clearing a stale marker whose server already died.

**ASR1 (P1): post-ASR glossary / custom vocabulary.** [DONE] Recurring proper nouns, client and product names, and en/uk code-switched terms are mangled by ASR every meeting, and the damage propagates into summaries, embeddings, ask, and roster display. Acceptance: a glossary term is normalized consistently across transcript, summary, and the embedding index; per-workflow entries apply only to that workflow's meetings; zero egress impact (a pure local transform).

Acceptance: a hotkey press mid-recording lands a marker in the sidecar; the chip seeks correctly; a flagged moment demonstrably surfaces in the summary; works under regulated/NDA; visual eyeball owner-owed.

**AI6 (P2): recurring-series continuity.** [DONE] Summaries of recurring meetings re-extract known context and lose the thread between sessions. The first meeting in a workflow is a clean no-op.
