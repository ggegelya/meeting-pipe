**[DONE] PERF7 (P3): event-log rotation / size cap.** Shipped 2026-07-11. Original scope in the ToC row above. Rename rotation, chosen over ring-buffer trim for concurrency safety: atomic renames mean a writer that loses the race just appends to a fresh file, where an in-place newest-half rewrite could clobber a concurrent append (`pipeline_events.jsonl` has multiple concurrent writers). A file reaching `MEETINGPIPE_LOG_MAX_BYTES` (default 5 MiB) rotates `foo.ext` -> `foo.1.ext` (the index goes before the extension so a rotated file stays valid JSONL / a tail-able log), shifting older generations up and dropping past 3 backups, so each log family self-bounds at ~4x the cap. Swift: `Log.rotateIfNeeded` at the single `appendData` choke point covers `events.jsonl` + `main.log`/`daemon.log`/`recorder.log`, and `PipelineLauncher.runMP` rotates `pipeline.log` before opening its per-run handle. Python: `events.rotate_if_needed` in `emit` (for `pipeline_events.jsonl`) and reused by `orchestrate._configure_logging` before it opens `pipeline.log` (both sides rotate at open time, so a daemon-spawned `mp` never has a stale open handle to a renamed file). Readers span the boundary via `Log.logGenerations` / `events.log_generations` (base plus generations, oldest first): `mp logs`, `mp analyze-detection` (its `pair_sessions` sorts by `ts`, so cross-file order is moot and a session split across a rotation still pairs), and the two stdlib-only scripts (`dogfood_report.py`, `valid1_check.py`) which get a local mirror of the generation glob rather than importing the package (they advertise stdlib-only). `launchd.{out,err}.log` are launchd-owned and outside the app's write paths, so out of scope. Docs corrected in passing: ARCHITECTURE + README listed `detector.log` / `recording.log`, which are never written (only the `main`/`daemon`/`recorder` categories reach `Log.writeLine`). First-ever tests over the log write path on both sides (4 Swift `LoggerTests` incl. generation-bounding, 7 `test_events.py` incl. a session paired across a rotation boundary). Verified: daemon 1138 green, pipeline 680 green (ruff + pyright clean). No detection or behaviour change.
