## Task specs

**[DONE] PIPE1 (P1): honest publish-failure contract (AUD-14).** Shipped 2026-07-10; the original spec text is the `**PIPE1` entry in [`q4-final.md`](./q4-final.md) (carried from Q4 unchanged). As built: `publish_router.all_sinks_failed` distinguishes "every sink failed" from "no sink ran" (a regulated run whose only sink was Notion is a clean success, not a failure), and `EXIT_PUBLISH_FAILED = 3` is returned by `run-all`, `mp publish`, and `mp publish-from-paste`. `orchestrate` emits `run_failed` with `stage=publish` instead of `run_completed`. Deviation on "resolve the page URL from per-sink sidecars": per-sink sidecars cannot express a run's outcome, because a publisher that raises never writes one and the previous run's `<stem>.notion.json` survives, indistinguishable from a fresh one. `fanout` therefore writes a run-scoped `<stem>.publish.json` (state + landed page URL + per-sink detail, rewritten every run) and Swift reads it through `PublishResult.load`; this also fixes AUD-30's nil-success on a non-Notion workflow and the page-less-sink-ahead-of-Notion bug. `SinkDispatcher.stage(for:)` maps exit 3 to a new `PipelineFailureSidecar.Stage.publish`, whose `retriesFromSummary` drives the stage-aware `retryMeeting`: a publish failure republishes the summary already on disk instead of paying the summarizer twice. Scope added beyond the spec, both load-bearing for the acceptance criteria: `mp publish-from-paste` inherits the same exit contract (the daemon's in-app paste box read its zero exit as "published"), and `republishMeeting` now writes a stage=publish failure sidecar on failure, without which a failed republish left the row looking done and Retry was never offered.

**[DONE] SEC13 (P1): make zero-egress literal at the process boundary.** The guard is honest at the sink layer (armed at all ten sink-capable entry points, verified) but not literal as documented. Two material gaps: (1) it patches in-process httpx only, while `LocalSummaryClient._spawn` launches `mlx_lm.server` with an unscrubbed env and `HF_HUB_OFFLINE` appears nowhere in the repo, so a regulated/NDA run with an uncached model downloads it from huggingface.co from inside the child, and huggingface_hub / mlx_embeddings use the requests stack outside the patch; (2) the daemon's SEC5 token strip is knowingly undone by `load_secrets()` refilling from the Keychain, collapsing defense-in-depth to the single httpx layer, with the refilled tokens inherited by the server child. Scope: (a) when the guard is armed, spawn local children with a scrubbed env (drop `ANTHROPIC_API_KEY` / `NOTION_TOKEN` / `HF_TOKEN`) plus `HF_HUB_OFFLINE=1` and `HF_HUB_DISABLE_TELEMETRY=1`; an uncached model under regulated then fails closed with an actionable message pointing at `mp prefetch-model` (fail-closed beats silent egress); (b) `load_secrets()` skips the cloud-token Keychain refill when `egress_guard.is_armed()` (every entry point arms before loading, so the ordering already holds); (c) route `publish_notion.py`'s hand-rolled regulated skip through the `effective_sinks` chokepoint so the invariant keeps a single owner (the TECH-ARCH1 promise); (d) extract the `apply_workflow_overrides` + `arm_for_config` + `load_secrets` triple, copy-pasted at 7+ entry points, into one shared entry helper so a future entry point cannot forget the clamp; (e) correct the `config.py` "no transcript can leave the machine" wording: the LAN sink ships summaries and transcripts to a mount by design, so the honest claim is "no cloud egress". Acceptance: under regulated/NDA the mlx server env carries no cloud tokens and no HF fetch occurs (asserted via a fake spawn in tests); an uncached-model regulated run fails with the prefetch message; all entry points route through the shared helper; the existing egress tests stay green. Shipped 2026-07-10. Deviation on (c): `publish_notion`'s skip routes through the new `config.zero_egress(cfg)` predicate rather than `effective_sinks`, because `mp publish-notion` names its sink explicitly and gating it on the configured sink list would silently no-op the command for an obsidian-only install. Same single-owner outcome (the predicate now backs `effective_backend`, `effective_sinks`, `arm_for_config`, `workflow.apply_overrides`, and the publish_notion skip), no behaviour change. `arm()` scrubbing `os.environ` process-wide, rather than only building a clean child env, was added on top of the spec: mlx_embeddings reaches huggingface_hub in-process over `requests`, outside the httpx patch, so the env is the only lever that covers it.

**STOR1 (P1): audio library storage governance.** [DONE] Stereo WAV is ~0.7 GB per hour and `raw/` grows unbounded; MIC13 covers only the `originals/` recovery copies and PERF7 only the event logs. Scope: a per-workflow audio retention policy (keep forever / transcode to FLAC or AAC N days after a terminal published-or-skipped state / drop audio keeping transcript + summary), a reaper sweep sharing MIC13's mechanism (one reaper, two scopes), a Preferences Storage section (library size, per-policy breakdown, HuggingFace model-cache size with an evict-unused-models control, waveform-cache size), and disk numbers in `mp doctor`. Consumers must survive a transcode: playback and the waveform peaks re-derive from the new file (the cache is keyed on size + mtime, so it self-invalidates) and the Library's audio affordances follow the extension. Defaults are conservative: every workflow starts on keep-forever until the owner opts it in, and a policy never touches a non-terminal meeting (`Needs you` members are exempt). Acceptance: a meeting past its window is transcoded or reaped on the next sweep with playback and waveform intact; delete cascades stay correct against MIC13; Preferences and doctor report real sizes; keep-forever workflows are byte-unchanged.

**SEC12 (P1): iCloud/cloud-sync egress check on the library path.** [DONE] The default `~/Documents/Meetings/` sits inside iCloud's Desktop & Documents scope when that setting is on, so every WAV and transcript silently uploads to Apple, an OS-level hole in the zero-egress claim; ADR 0016 already mandates iCloud/Time-Machine exclusion for `originals/`, but the main library has no check. Scope: detect a cloud-synced library root (iCloud Desktop & Documents state, plus Dropbox / Google Drive / OneDrive path patterns), WARN in `mp doctor` and in a Preferences Storage status line naming the provider, and hard-FAIL the doctor check under `regulated_mode` or when any NDA workflow exists; offer an assisted move to a non-synced root (guided, never performed without explicit confirmation) or a per-folder exclusion where the provider supports one. README documents the caveat. Acceptance: a synced library path is flagged with the provider named; regulated + synced is a FAIL, not a note; the assisted move yields a working library at the new root with config updated.

**ASR1 (P1): post-ASR glossary / custom vocabulary.** [DONE] Recurring proper nouns, client and product names, and en/uk code-switched terms are mangled by ASR every meeting, and the damage propagates into summaries, embeddings, ask, and roster display. Scope: a deterministic substitution layer applied at transcript finalize in the pipeline, before summarize/embed ever see the text: a global glossary plus an optional per-workflow table (TOML under `~/.config/meeting-pipe/`, the workflows precedent), case-insensitive whole-word matching that is Unicode-aware (Cyrillic word boundaries), longest-match-first, with applied-substitution counts logged to pipeline events. Transcripts stay write-once: the glossary applies to new meetings at finalize; re-applying to an old meeting is an explicit regenerate, never a mutation. An optional bounded-edit-distance fuzzy stage for names ships behind a flag, off by default (a wrong substitution is worse than a missed one). Any new `uk` entries are verified with the user (the locale note in the delegation section). Acceptance: a glossary term is normalized consistently across transcript, summary, and the embedding index; per-workflow entries apply only to that workflow's meetings; zero egress impact (a pure local transform).

**FEAT8 (P1): in-meeting moment flag.** [DONE] The one in-meeting capability the competitor set has that meeting-pipe lacks, and it fits the register: one global hotkey (configurable, chosen in-task alongside `manual_hotkey`) during a recording stamps a timestamp marker; the HUD acknowledges with a single quiet blink (opacity-only, reduced-motion honoured), no persistent chrome. Markers are written at stop as `<stem>.markers.json` (schema_version stamped, documented in CONVENTIONS, the DOC6 precedent); the transcript tab renders each marker as an anchor chip at its timestamp (click = seek, the existing play-head path); summarize receives the flagged excerpts (the segments spanning each marker) as user-flagged moments with an instruction to reflect them, so capture is LLM-free and deterministic while emphasis is model-side. BYO and long-meeting paste bundles include the marker list. No new egress class: flagged excerpts travel to whichever backend already summarizes the transcript. Acceptance: a hotkey press mid-recording lands a marker in the sidecar; the chip seeks correctly; a flagged moment demonstrably surfaces in the summary; works under regulated/NDA; visual eyeball owner-owed.

**STOR2 (P1): backup + restore manifest.** [DONE] The compounding assets are irreplaceable and scattered: the library (`raw/` + sidecars + `digests/`), `~/.config/meeting-pipe/` (config, workflows, voiceprint, roster, glossary), the corrections corpus in `~/Library/Application Support/MeetingPipe/`, and the Keychain items; caches are rebuildable and excluded. Scope: `mp backup <dir>` writes a dated archive of the non-rebuildable roots plus a manifest naming the Keychain items to re-create (secret values are never exported); `mp restore <archive>` unpacks to the configured roots (per ADR 0003 a manual copy also works, and the runbook says so); `mp doctor` reports last-backup age when a backup marker exists; a README migration runbook (new Mac: install, restore, re-add the Keychain items, done). Acceptance: a backup restored into a scratch HOME yields a working library, workflows, roster, and corrections state; the manifest names every Keychain item; doctor reports backup age; the runbook is verified by walking it.

**AI6 (P2): recurring-series continuity.** [DONE] Summaries of recurring meetings re-extract known context and lose the thread between sessions. Scope: at summarize time, inject the same workflow's previous meeting as a bounded context block (its decisions plus its still-open actions, post-AI1 resolved state), with instructions to mark carried-over items as carried over rather than re-extracting them as new discoveries. The workflow is the series key (explicit, already resolved per meeting); no fuzzy series inference. Prompt-only in v1, no schema change; the block is bounded (~1K tokens, latest prior meeting only) and assembled from disk, so cloud and local backends get it identically and the egress profile is unchanged. The first meeting in a workflow is a clean no-op. Acceptance: the second meeting in a workflow produces a summary that acknowledges the carried-over open action instead of re-listing it as new (dogfoodable via `mp dogfood`); regulated/NDA behaviour unchanged.

**LOCAL7 (P2): local summaries honor summary_language on non-English meetings.** [DONE] The engine comparison found the local backends ignore `summary_language = auto` on Ukrainian: the 7B drifts uk to en, the 14B uk to ru; only Anthropic preserves Ukrainian. Strengthen the local language directive (`summarize_local` plus the Apple Intelligence instructions) and add a cheap detected-language-mismatch retry so a uk transcript yields a uk (or the configured-language) summary. Shipped: `summarize_local.py` reinforces the resolved target right before the transcript and, after the fact, detects the summary's language (a tiny script + uk/ru-signature detector) and replays once when it disagrees; the target is threaded from `summary_language` and, for `auto`, from a Cyrillic read of the transcript, with non-verifiable Latin codes (de, es, ...) left untouched so no bogus retry fires. `AppleIntelligenceSummarizer.languageDirective` was strengthened to the same field-by-field, non-negotiable form. On-device MLX efficacy on real Ukrainian meetings is owner-owed dogfood (the fix is unit-tested structurally).

**LOCAL8 (P2): Apple Intelligence, experimental gate + viability fixes.** [DONE] The engine comparison showed the macOS 26 Apple Intelligence backend is native/free/zero-egress but not usable as a daily summarizer today: its 4096-token context forces heavy chunking (minutes/meeting), it echoes raw dialogue on short meetings, ignores the bullet cap (~11 vs 5), mislabels Ukrainian, and 1/5 failed. Keep the code but mark it experimental / not-recommended in Preferences (so it is not silently chosen), and track the fixes that make it viable: LOCAL3 (reduce + Cyrillic window, shipped in Q5), LOCAL7 (language), bullet-cap adherence, the short-meeting transcript-echo. Revisit-when: those land OR Apple widens the system-model context. Owner kept it (not dropped) as the only native zero-egress path and the I7 trigger. Shipped: the Preferences backend picker labels it `Apple (experimental)` and the footer states plainly it is experimental / not recommended for daily use with the caveats; the instruction/reduce prompts now cap summary bullets at 5 and forbid echoing raw dialogue; LOCAL3 and LOCAL7 already landed, so the only open revisit condition is Apple widening the system-model context. On-device efficacy stays owner-owed dogfood (SUM1-APPLE precedent).

**UX18 (P2): in-app paste completion for BYO / long-meeting bundles.** [CLOSED 2026-07-10, not implemented] The 2026-07-10 architecture review found the premise stale: `SummaryTab.swift` already renders an in-app "Paste your summary" editor with inline validation errors and a "Save & publish" action wired through `publishFromPaste` (TECH-UX3, its comment reads "no terminal command"), covering both BYO and long-meeting rows; the egress guard and workflow overlay ride the same publish-from-paste path the spec asked for. The remaining delta against the spec below (a sheet presentation instead of the inline pane) was judged not worth a task. Original spec for the trail: the paste-pending detail pane gains "Paste summary...", opening a sheet (raw markdown box, parsed through publish-from-paste's existing parser into the typed model, with inline validation errors); on confirm it publishes through the PIPE2 fanout with the workflow overlay applied and the egress guard armed, exactly like the CLI path, which stays supported. DSN22's "Reveal bundle" verb remains the manual escape hatch.
