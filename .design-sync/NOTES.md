# design-sync notes for meeting-pipe

- This repo is not a JS package. The design system is `design/colors_and_type.css` (the `--mp-*` tokens) plus `design/ui_kits/macos_app/*.jsx`, a Babel-in-browser kit whose files share one global lexical scope (unique top-level const names are the kit's own invariant; see the kit README).
- `design/package.json` is a NOMINAL manifest that only anchors the converter's PKG_DIR and version. It is never npm-installed and must stay dependency-free.
- The bundle entry is generated by `.design-sync/make-entry.mjs` (`cfg.buildCmd`) into `design/.ds-entry.jsx` (gitignored): kit files concatenated in `index.html` script order, `window.X = X` script-tag exports stripped, `"../../assets/*"` path strings rewritten to dataurl imports, `export { ... }` appended from `componentSrcMap` keys. Adding a component = add it to `componentSrcMap` (+ `dtsPropsFor`), nothing else.
- `macos-window.jsx` (MacWindow, MacGlass, traffic lights) is deliberately EXCLUDED: not loaded by the kit's `index.html`, hardcoded non-token colors (a "macOS Tahoe" glass kit from the initial design commit). User confirmed exclusion on 2026-07-05. Revisit only if the DSN20-29 redesign adopts liquid glass.
- Fonts: Inter Tight 500/600/700 and JetBrains Mono 400/500 vendored from Google Fonts (both OFL; user approved 2026-07-05) into `.design-sync/fonts/` (committed, so re-syncs never re-download). SF Pro / `-apple-system` stays a system-font fallback by design (Apple license); off-Mac renderers show Helvetica-ish instead, accepted.
- Converter deps plus `react@18.3.1` + `react-dom@18.3.1` (pinned to match the kit page's CDN versions) live in `.ds-sync/node_modules`; every converter invocation passes `--node-modules .ds-sync/node_modules`.
- `Icon` is kit-global: most components reference it bare. In authored previews, import `Icon` alongside the component under test.

## Preview authoring (folded from the 2026-07-05 fan-out)

- HUD components (MeetingPrompt, MenuBarDropdown) are translucent and wash out on white; their previews wrap each cell in a dark desk gradient (`linear-gradient(180deg, #2C3037 0%, #14161A 100%)`). SL* and PW* leaves are transparent and get a light paper wrapper (`--mp-bg`, padding 16); `SLEmpty` needs a fixed ~320px-high wrapper because its root is `height:100%`.
- Composition wrappers the leaves need (a menu is a 240px column; PW rows live inside `PWGroup`; `PWSlider` renders as fragments that want a stacked 380px label row) are baked into the authored `.tsx`, so they survive re-sync.
- `Icon` returns `null` for an unknown `name`; the 47 valid names are exactly the `Icons.jsx` switch cases and match `cfg.dtsPropsFor.Icon` 1:1. `dtsPropsFor` is the authoritative prop source for every component (the kit ships no real `.d.ts`), so keep it in step with the sources.
- `overrides.{Notification,MeetingPrompt,PreferencesWindow,SummaryLibrary}.cardMode = "column"` (config) stops the wide surfaces from being cropped in the product's grid card. `Notification` tripped `[GRID_OVERFLOW]` (wide) before this; the other three are wide surfaces by construction.

### Known render warns (triaged, expected on re-sync)
- `[FONT_MISSING] "SF Pro Text", "Inter", "SF Pro Display"` — intentional. SF Pro is Apple-system-only (never shipped); the CSS `--mp-font-sans` stack lists bare `"Inter"` as a fallback we do not vendor (we ship **Inter Tight**, the display face, not Inter). Both are deliberate system/fallback references, not missing brand fonts. Do not "fix" by vendoring Inter.
- `▲ Duplicate key "fontSize"` from esbuild on `Notification` — the kit's `Notification.jsx` sets `fontSize` twice in one inline-style object (last wins, intentional mono-vs-base switch). Cosmetic; lives in the kit source.
- SummaryLibrary's capture sheet crops its 1120px+ multi-state gallery to the first frame; the rendered content grades good and `cardMode:column` gives it full card width in-product. Not a defect.

### Re-sync risks
- Fonts are vendored once into `.design-sync/fonts/` and committed; a re-sync never re-fetches. If a weight is added to `colors_and_type.css`, re-run the font fetch (see the scratch `fetch-fonts.mjs` recipe in git history / conventions) and re-vendor.
- Previews import from `'meeting-pipe-design'` (the nominal `cfg.pkg`). If the package is ever renamed, every `.design-sync/previews/*.tsx` import must change too.
- The bundle is a concatenation of kit files in `index.html` script order via `make-entry.mjs`. If a NEW top-level `const` name collides across kit files (the kit's one-global-scope invariant), the bundle breaks the same way the kit page would; keep top-level const names unique.
- `dtsPropsFor` is hand-maintained. A prop added/removed in a kit source will NOT show up in the design agent's contract until `dtsPropsFor` is updated to match.
